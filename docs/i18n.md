# Internationalization (i18n)

The AI Tamer plugin is fully prepared for translation. All user-facing strings are wrapped in WordPress standard translation functions (`__()`, `esc_html_e()`, etc.) using the `ai-tamer` text domain.

## Setup
The plugin is configured with:
- **Text Domain**: `ai-tamer`
- **Domain Path**: `/languages`
- **Loading**: `load_plugin_textdomain()` is called on the `init` hook in `AiTamer\Plugin`.

## Translation Workflow

To create translations, follow these steps:

### 1. Generate the POT File (Template)
The `.pot` file contains all the "master" strings from the code. You can generate it using:
- **WP-CLI**: `wp i18n make-pot . languages/ai-tamer.pot`
- **Poedit**: Go to `File > New from source code` and scan the plugin folder.

### 2. Create Language-Specific PO Files
Create a copy of the `.pot` file for each language you want to support. The filename must follow this pattern:
`ai-tamer-{locale}.po`

Example for Spanish (Spain): `ai-tamer-es_ES.po`

### 3. Compile to MO Files
Once you finish translating in your `.po` file, it must be compiled into a `.mo` file (which WordPress actually reads).
- Tools like **Poedit** do this automatically when you save.
- You can also use the **Loco Translate** plugin directly in the WordPress dashboard to manage these files.

## Summary of Files
All translation files should be placed in the `/languages` directory of the plugin:
- `ai-tamer.pot` (Optional template)
- `ai-tamer-es_ES.po` (Translations for Spanish)
- `ai-tamer-es_ES.mo` (Compiled for Spanish)
